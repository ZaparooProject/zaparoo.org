## Launch Guard

When a game is already running, Zaparoo now holds incoming scans instead of launching immediately. A sound plays when the staged scan is ready, and a deliberate second tap confirms the switch. If nothing happens within 15 seconds, the scan is discarded and the running game continues uninterrupted.

This solves a common annoyance: cards accidentally left on readers, bumped setups, or children enthusiastically scanning everything in reach.

Enable it in `config.toml`:

```toml
[readers.scan.launch_guard]
enabled = true
timeout = 15
delay = 2
```

The `delay` setting adds a mandatory wait before the second tap is accepted, so an impatient double-tap can't bypass it. There's also a `require_confirm` mode where only a physical button or API call can confirm a switch; re-taps do nothing. The docs include a Python example for wiring a GPIO button on a Raspberry Pi as a confirm button.

## PixelCade

PixelCade LED matrix displays now work as a publisher in Zaparoo. When a game starts, Zaparoo sends the system and title to PixelCade's local API and the matching artwork appears on the display. About 70 systems are mapped out of the box, with a lowercase fallback for anything not in the list.

Configure it in `config.toml`:

```toml
[[publishers.pixelcade]]
host = "192.168.1.x"
mode = "arcade"
on_stop = "default"
```

The `on_stop` field controls what happens when the game stops: `blank` clears the display, `default` shows the PixelCade default marquee, or leave it out to do nothing.

## RePlayOS

[RePlayOS](https://replayos.net) on Raspberry Pi 4 is now fully supported. Zaparoo tracks active games by watching which libretro cores are loaded, captures screenshots using the RePlayOS keyboard shortcut, and installs as a systemd service with resource limits so it never competes with the emulator for CPU or I/O.

## New ZapScript commands

**`**launch.last`**: re-launches the most recently played game. Add a number to step further back in history:

```
**launch.last
**launch.last:2
**launch.last:3
```

Handy for a dedicated "last played" card, or a physical button that cycles through recent sessions.

## Other changes

- **Reader enable/disable**: individual reader connections can be enabled or disabled without removing them from config. The TUI shows "(disabled)" next to disabled entries
- **Beta update channel**: set `update_channel = "beta"` in config to receive pre-release builds through the built-in updater
- **Audio volume**: new `[audio] volume` setting (0-200, default 100). Values above 100 amplify the feedback sounds. Configurable without a restart via the settings API
- **ZapScript security hardening**: new config options to block specific commands by name (`block_commands`), restrict which URLs the `http.get`/`http.post` commands can reach (`allow_http`), and control which keyboard inputs are allowed through (`input_restriction`)
- **Encrypted WebSocket connections**: optional PAKE-based pairing with a 6-digit PIN and AES-256-GCM encryption. Off by default. See the [API docs](/docs/next/core/api/encryption) for setup
- **PN532 I2C auto-detection**: PN532 readers connected over I2C are now detected automatically without manual config
- **Server-Sent Events**: new `/api/v0.1/events` endpoint as an alternative to the WebSocket notification stream, useful for integrations that prefer SSE
- **Media indexing pauses during gameplay**: the background media scanner pauses while a game is running and resumes when it stops, cutting I/O contention during sessions
- **Tag database expanded**: dozens of new tags from the GameDataBase project for arcade boards, input devices, and more. Old deprecated tag names on existing cards resolve automatically, so nothing breaks
- **MiSTer: DualRAM alt launchers**: support for 3DO, Jaguar, PSX, and Saturn using the [Dual RAM Console Cores](https://github.com/TheJesusFish/Dual-Ram-Console-Cores) builds, which go in `_Console (Dual SDRAM)/`
- **MiSTer: `load_path` config override**: pin a specific RBF file for any launcher, useful when multiple cores share the same short name (such as the three N64 cores across different folders)
- **New API methods**: `confirm` (launch guard confirmation), `clients` and `clients.delete` (paired client management). See the [API docs](/docs/next/core/api/methods)

## Performance

- **CPU usage fixed on multi-core systems**: a misconfigured Go memory limit was causing garbage collection to consume up to 50% of available CPU on high-core-count machines in some cases. Removed in favour of default Go GC behaviour
- **Feedback sounds are 12x smaller**: audio files converted from WAV to OGG Vorbis, cutting the embedded binary size considerably

## Bug fixes

- Fixed sensitive input sequences (such as passphrases) being written to debug logs in plaintext
- Fixed media index hanging on resume from a previously interrupted indexing run
- Fixed ZapLink token size limit not being enforced correctly
- Fixed hold mode with RetroBat and relative config paths on Windows
- Fixed LLAPISuperGrafx launcher not handling `.sgx` files on LLAPI cores
- Fixed Saturn MGL timing on some cores
- Fixed Windows system tray menu items not opening correctly
- Fixed macOS `.DS_Store` and other system files causing noise during indexing
- Fixed several URI decoding edge cases involving `?`, `#`, and literal `%` characters
